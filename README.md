# 任务管理系统

基于 Manul 编程语言开发的功能完整的任务管理系统，支持项目管理、任务分配、进度跟踪和团队协作。

## 主要功能

### 用户管理
- 用户注册、登录、认证
- 多角色权限系统（管理员、项目经理、开发者、测试员、查看者）
- 用户资料管理
- 会话管理和令牌刷新

### 项目管理
- 创建和管理项目
- 项目状态跟踪（规划中、进行中、暂停、已完成、已取消）
- 项目成员管理和角色分配
- 项目优先级设置
- 项目进度统计

### 任务管理
- 创建和分配任务
- 任务状态流转（待办 → 进行中 → 审核中 → 测试中 → 已完成）
- 任务优先级管理
- 工时记录和估算
- 任务评论和附件
- 到期提醒和逾期跟踪

### 数据统计
- 个人任务统计面板
- 项目进度报告
- 团队工作统计
- 逾期任务提醒

## 系统架构

### 领域模型 (src/domain/)
- `User` - 用户实体，包含认证和权限管理
- `Project` - 项目实体，支持成员管理
- `Task` - 任务实体，支持评论和附件
- `Session` - 会话管理

### 枚举类型
- `UserRole` - 用户角色（管理员、经理、开发者等）
- `ProjectRole` - 项目角色（所有者、管理员、成员、查看者）
- `ProjectStatus` - 项目状态
- `TaskStatus` - 任务状态
- `Priority` - 优先级

### 服务层 (src/service/)
- `AuthService` - 认证服务
- `UserService` - 用户管理服务
- `ProjectService` - 项目管理服务
- `TaskService` - 任务管理服务
- `DashboardService` - 数据统计服务

### 值对象 (src/value/)
- `DashboardStats` - 仪表板统计数据
- `ProjectProgress` - 项目进度信息
- `TeamStats` - 团队统计数据

## API 功能

系统自动生成 REST API 接口，支持：

- **认证相关**: 登录、登出、令牌刷新、密码修改
- **用户管理**: 创建用户、更新资料、角色管理
- **项目管理**: 项目 CRUD、成员管理、状态更新
- **任务管理**: 任务 CRUD、状态流转、工时记录、评论管理
- **数据统计**: 个人面板、项目进度、团队统计

## 权限控制

### 系统级权限
- 管理员：可管理所有用户和项目
- 项目经理：可创建和管理自己的项目
- 其他角色：只能参与分配给自己的项目

### 项目级权限
- 项目所有者：完全控制项目
- 项目管理员：管理项目和成员
- 项目成员：创建和编辑任务
- 查看者：只读访问

### 任务级权限
- 任务创建者可编辑任务
- 任务分配者可更新状态和记录工时
- 项目管理员可管理所有任务

## 部署说明

本项目基于 Manul 编程语言开发，部署到 Miaobu 平台后会自动：

1. 编译 `.mnl` 源文件
2. 生成 REST API 端点
3. 初始化数据库和示例数据
4. 启动 Web 服务

访问部署后的 URL 即可使用 API 接口进行项目和任务管理。
